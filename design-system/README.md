# YSWORKS Production Design System

- Status: governed implementation contracts
- Scope: current Public Website and explicitly approved shared foundations

## Authority

The [Company Bible](../docs/COMPANY_BIBLE.md) defines company personality and
the governing design, motion, and user-experience philosophy
(*I.VIII; I.XIV–XV*). The subordinate
[Brand Bible](../docs/BRAND_BIBLE.md) is the constitutional identity authority
for visual language, motion, and interfaces. The
[Approved Design Authorities](../docs/design/README.md) map the Founder-approved
operational Brand Book, UX Foundation, Design System, and Design Tokens to their
canonical sources.

Files in this directory standardise implementation. They do not create colours,
company identity, product architecture, or cross-product runtime decisions.

## Product Boundary

The Public Website, future Client Workspace, and private YS AI OS may share an
approved visual family without sharing authentication, authorisation, data,
navigation, or runtime. A Public Website implementation decision does not
automatically govern another product.

## Maintenance

Update these contracts when an approved design decision changes or repeated
implementation evidence justifies a semantic role. Keep values in their
canonical token source, cross-reference higher authority, and remove superseded
rules rather than maintaining parallel variants.
