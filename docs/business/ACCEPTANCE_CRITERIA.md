# Acceptance Criteria

## Purpose

These criteria define when YSWORKS Web is ready to operate as a public commercial
presence.

## Repository

- [ ] Documentation is indexed and easy to find.
- [ ] No private system details are present.
- [ ] No secrets or private environment files are committed.
- [ ] CI passes.
- [ ] Documentation links are valid.

## Website Structure

- [ ] Navigation links only to existing pages or sections.
- [ ] First-release pages exist: Home, Services, About, Contact, Privacy.
- [ ] Work, Labs, and Journal are added only when real public content exists.
- [ ] Footer includes useful navigation and legal links when available.
- [ ] No placeholder proof is presented as credibility.

## SEO

- [ ] Production canonical URL is configured.
- [ ] Page titles and descriptions are unique.
- [ ] Open Graph metadata is present.
- [ ] `robots.txt` exists.
- [ ] Sitemap exists.
- [ ] Internal links are valid.

## UX

- [ ] Primary CTA requests a conversation.
- [ ] Contact path asks for a brief problem description and sets expectations.
- [ ] Mobile navigation is usable.
- [ ] Heading hierarchy is valid.
- [ ] Content is readable on mobile and desktop.

## Accessibility

- [ ] Keyboard navigation works.
- [ ] Focus states are visible.
- [ ] Contrast is sufficient.
- [ ] Semantic HTML is used.
- [ ] Reduced motion is respected.

## Performance

- [ ] Pages are static-first where possible.
- [ ] Client-side JavaScript is minimal.
- [ ] Assets are optimized.
- [ ] Layout shifts are avoided.
- [ ] Build output is reviewed.

## Commercial Readiness

- [ ] Services are clear and public-safe.
- [ ] Contact expectations are clear.
- [ ] Portfolio avoids fake clients or fake results.
- [ ] Labs and products are not presented as completed if exploratory.
- [ ] YS AI OS remains private.
- [ ] Canarias Tourism Platform is not presented as launched until public
  material is approved.

## Form Readiness

Before publishing a form:

- [ ] Privacy page is ready.
- [ ] Turnstile is configured.
- [ ] Honeypot exists.
- [ ] Rate limiting exists.
- [ ] Server-side validation exists.
- [ ] Data collection is minimal.
