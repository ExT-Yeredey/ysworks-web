# YSWORKS Documentation Navigation

## Purpose

This directory is the maintained navigation layer for the public-safe YSWORKS
repository. It helps readers locate authority and supporting material without
duplicating the documents themselves.

Navigation files are descriptive indexes. They do not create company,
architectural, product, security, or implementation authority.

## Start Here

| Need | Navigation document |
| --- | --- |
| Find a document or repository area | [Document Map](DOCUMENT_MAP.md) |
| Understand systems and architectural contracts | [Architecture Map](ARCHITECTURE_MAP.md) |
| Resolve governing precedence | [Authority Map](AUTHORITY_MAP.md) |
| Understand document and implementation dependencies | [Dependency Map](DEPENDENCY_MAP.md) |
| Check an official term | [Glossary](GLOSSARY.md) |
| Expand an acronym or abbreviation | [Acronym Index](ACRONYM_INDEX.md) |

The primary documentation entry point remains the
[YSWORKS Documentation Index](../README.md).

## Maintenance

- Add a document to the applicable map when it is created, renamed, moved, or
  retired.
- Link to the canonical source rather than copying its rules.
- Mark historical filenames and absent planned documents explicitly.
- Keep private topology, secrets, client data, workflows, and internal records
  out of this public navigation layer.
- Treat a broken or ambiguous navigation path as a documentation defect.
