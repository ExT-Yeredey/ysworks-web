# Branding

## Purpose

This document defines the public brand direction for YSWORKS Web.

It is not a marketing script. It is a guide for consistent public presentation.
The [Brand Bible](../docs/BRAND_BIBLE.md) is the constitutional identity
authority. This document applies it to the Public Website and does not create
independent brand doctrine.

## Brand Personality

YSWORKS should feel:

- Precise.
- Calm.
- Technical.
- Trustworthy.
- Independent.
- Modern.
- Serious without being cold.

The brand should feel like an engineering studio that values durable systems, clear thinking, and careful execution.

## Desired Perception

Visitors should perceive YSWORKS as:

- Professional.
- Competent.
- Security-aware.
- Performance-minded.
- Clear about public boundaries.
- Focused on practical outcomes.

The website should not overexplain internal methods. It should communicate quality through restraint and consistency.

## Brand Values

- Engineering quality.
- Clarity.
- Security.
- Performance.
- Maintainability.
- Accessibility.
- Honest communication.
- Respect for privacy.

## Visual Direction

The visual direction should be minimal, dark-first, precise, and content-led.

Use:

- Strong typography.
- High-contrast neutrals.
- Subtle accents.
- Clean layouts.
- Purposeful spacing.
- Minimal motion.
- Original visual assets.

Primary 500, `#6D5EF7`, is the official YSWORKS brand color. The complete
cross-product palette and usage contract is
[YSWORKS Design Tokens v1.0](../docs/design/YSWORKS_DESIGN_TOKENS_V1.md).

Avoid design choices that feel decorative before they feel useful.

## Emotional Goals

The website should create:

- Confidence.
- Calm.
- Focus.
- Trust.
- Technical credibility.

It should not create pressure, spectacle, or confusion.

## What YSWORKS Should Never Look Like

YSWORKS should never look:

- Flashy.
- Generic.
- Noisy.
- Gimmicky.
- Like a template startup landing page.
- Like a fake enterprise consultancy.
- Like an AI hype product.
- Like a stock-photo agency site.
- Like a dashboard pretending to reveal private systems.

The brand should not depend on fake people, fake offices, fake customer logos, invented metrics, or exaggerated AI claims.
