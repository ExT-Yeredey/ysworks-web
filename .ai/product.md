# Product

## What This Website Is

YSWORKS Web is the public website for YSWORKS.

It is a public communication surface for:

- Studio identity.
- Public positioning.
- Public service context.
- Contact or inquiry entry points.
- Public documentation about the website repository.

It should demonstrate engineering quality through implementation, performance, accessibility, and restraint.

## What This Website Is Not

This website is not:

- A private platform.
- A public view into internal systems.
- A product dashboard.
- A technical whitepaper for proprietary systems.
- A place to document internal workflows.
- A repository for private automation or business logic.
- A roadmap for confidential work.

## Public Objectives

The public website should:

- Explain what YSWORKS is at a high level.
- Build trust through clarity and quality.
- Load quickly.
- Work well on mobile and desktop.
- Be accessible.
- Be easy to maintain.
- Keep private systems private.

## Private Boundaries

The website must not expose:

- Internal architecture.
- AI agents.
- Prompts.
- MCP servers.
- Automation workflows.
- Raspberry infrastructure.
- Business logic.
- Internal documentation.
- Future confidential roadmap.
- Implementation details of internal systems.

Public content should stop at outcomes and high-level categories.

## Future Public Capabilities

Future public capabilities may include:

- A contact form.
- Public service pages.
- Public case studies only when real and approved.
- Public articles or technical notes.
- A lightweight public API layer for website needs.

Any future capability must remain public-safe and must not expose private systems.

## Product Quality Bar

Every public feature should be:

- Useful.
- Accessible.
- Fast.
- Secure.
- Maintainable.
- Honest.
- Easy to remove if it stops serving the site.
