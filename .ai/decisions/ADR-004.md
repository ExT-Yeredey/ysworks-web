# ADR-004: Keep YS AI OS Private

## Status

Accepted

## Context

YS AI OS is proprietary.

The public website repository exists only for the public YSWORKS website. It should not reveal internal architecture, AI agents, prompts, MCP servers, automation workflows, infrastructure, business logic, internal documentation, private roadmap items, or implementation details of internal systems.

## Decision

Keep YS AI OS private and out of this repository.

Public content may describe high-level outcomes or general public categories, but must not document private systems or operational details.

The public-safe responsibility and boundary in
[YSWORKS Enterprise Architecture](../../docs/YSWORKS_ENTERPRISE_ARCHITECTURE.md)
(*EA 6.1; EA 10.6*) do not expose or authorise the private system design. They
do not weaken this decision.

## Consequences

Developers and AI coding agents must treat public-safety as a core review requirement.

This means:

- No internal system diagrams.
- No prompt or agent documentation.
- No private workflow descriptions.
- No infrastructure disclosure.
- No confidential roadmap.
- No private business logic.
- No public code that depends on private implementation assumptions.

The public website can still communicate quality through speed, accessibility, security, writing, and maintainable code.

## Alternatives Considered

- Publish selected internal details: rejected because it creates unnecessary security and competitive risk.
- Use private systems as marketing proof: rejected because trust should not depend on exposing proprietary implementation.
- Maintain mixed public/private documentation in one repository: rejected because it increases the risk of accidental disclosure.
