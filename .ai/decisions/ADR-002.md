# ADR-002: Use Cloudflare Pages for Hosting

## Status

Accepted

## Context

YSWORKS Web should be simple to deploy, fast globally, and aligned with static output from Astro.

The hosting layer should not require custom server management for the public website.

## Decision

Use Cloudflare Pages as the hosting target for the public website.

## Consequences

Cloudflare Pages supports static site deployment, preview workflows, CDN delivery, and future public edge capabilities when needed.

This keeps the public website deployment model small:

- GitHub as source control.
- CI for validation.
- `pnpm build` for production output.
- `dist/` as the deploy artifact.

Any future Workers usage must be limited to public website needs, such as a contact form or lightweight public API layer.

## Alternatives Considered

- Vercel: strong developer experience, but Cloudflare Pages better aligns with the intended hosting target.
- Netlify: viable for static sites, but not the selected platform.
- Self-hosting: unnecessary operational complexity for this public website.
