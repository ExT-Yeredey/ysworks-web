# ADR-001: Use Astro for the Public Website

## Status

Accepted

## Context

YSWORKS Web is a public website, not a private application platform.

The site should be fast, accessible, static-first, easy to maintain, and capable of adding limited interactivity only where needed.

## Decision

Use Astro as the primary framework for the public website.

## Consequences

Astro supports static generation by default, keeps client-side JavaScript minimal, and allows interactive islands when a public feature requires them.

This aligns with the repository goals:

- Fast public pages.
- Minimal runtime complexity.
- Strong content and layout separation.
- Good TypeScript support.
- Maintainable component structure.

The team should avoid turning static content into client-rendered React without a clear user-facing need.

## Alternatives Considered

- Plain HTML and CSS: simple, but less scalable for structured components and future content.
- Full React application: more client-side JavaScript than this website needs.
- Next.js: capable, but heavier than required for the current static-first public site.
