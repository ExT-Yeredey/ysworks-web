# Glossary

## Purpose

This glossary defines public terminology for YSWORKS Web.

It avoids proprietary concepts and private system names.

## Engineering Studio

A team or practice focused on designing, building, and maintaining software systems.

Use this term to describe YSWORKS at a high level without implying a specific private operating model.

## Automation

Software behavior that reduces repetitive manual work.

Use this term only in public, general contexts. Do not describe private automation workflows or internal implementation details.

## Platform

A stable technical foundation that supports software products, services, or workflows.

Use carefully. Do not imply public access to private YSWORKS systems.

## Infrastructure

The hosting, runtime, networking, and operational foundation required to run software.

For this repository, public infrastructure refers only to website hosting and public deployment context.

## Cloud

Hosted computing services used to run, store, or deliver software.

For this website, Cloudflare Pages is the public hosting target.

## Workflow

A sequence of steps that moves work from intent to result.

Use only in general public language. Do not document internal workflows.

## AI

Artificial intelligence.

Use AI language to explain public outcomes only. Do not expose models, prompts, agents, internal tools, or proprietary system design.

## Integration

A connection between systems or services.

Public website integrations should be limited to public needs, such as analytics, forms, or hosting features, and should be reviewed for privacy and security.

## Static Site

A website generated into files that can be served without application runtime logic for each request.

This is the preferred model for YSWORKS Web when possible.

## Progressive Enhancement

An approach where the baseline experience works with HTML and CSS, and JavaScript adds improvements where useful.

This supports performance, accessibility, and resilience.

## React Island

A small interactive React component embedded in an otherwise static Astro page.

Use React islands only when interactivity is necessary.

## Public API

An endpoint intended for public website use.

Public APIs must not expose private systems, internal data, or implementation details.
